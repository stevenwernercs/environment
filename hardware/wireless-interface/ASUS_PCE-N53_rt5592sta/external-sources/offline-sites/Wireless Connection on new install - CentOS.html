    Hello All,

    I've just installed CentOS 7 on an old Acer laptop and am trying to get the wireless network working... During the install, the wizard was able to connect to my wireless network, and at conclusion of the installation the wireless settings were saved.

    I think that all the required software is installed, but when trying to bring up the wireless network, an error indicates that "No suitable device found for this connection."

    Code: Select all

    [root@Laptop]# nmcli connection show
    NAME   UUID   TYPE   DEVICE
    MyWifi <stuff> 802-11-wireless --
    enp2s0  <stuff> 802-3-ethernet  --  (not plugged in)

    [root@Laptop]# nmcli device status
    DEVICE  TYPE  STATE   CONNECTION
    enp2s0 ethernet unavailable --
    lo  loopback unmanaged --
    wlp3s0  wifi  unmanaged

    [root@Laptop]# nmcli connection up MyWifi
    Error: Connection activation failed: No suitable device found for this connection

    [root@Laptop]# nmcli device wifi rescan ifname wlp3s0
    Error: Device 'wlp3s0' is not a Wi-Fi device.

    [root@Laptop]# nmcli device show wlps30
    GENERAL.DEVICE:       wlp3s0
    GENERAL.TYPE:          wifi
    GENERAL.HWADDR:    00:22:FA:xx:xx:xx
    GENERAL.MTU           1500
    GENERAL.STATE          10 (unmanaged)
    GENERAL.CONNECTION:  --
    GENERAL.CON-PATH:    --
    IP4.GATEWAY:
    IP6.GATEWAY:


    I have reviewed other posts in this forum and confirmed that the Network Manager Wifi package is installed (from DVD)

    /var/log/dmesg shows that the wireless interface is at least detected by name (Intel(R) WiFi Link 5100 AGN, REV=0x54). Based on this I wanted to confirm that the device was successfully installed, plugged the laptop into a wired network and yum installed 'iwl5000-firmware' (using default repo) which turned out to already be installed with the latest versions.

    However, some commands that I've seen referenced in posts are not available (iw and ifconfig are most concerning)...

    I have plenty of networking experience, but my Linux CLI knowledge is still being cobbled together. Any help would be appreciated.

      Postby scottro » 2016/11/24 01:54:24

      Well, ifconfig worked quite well, so of course, Linux has deprecated it. One now has to install net-tools to get it.

      It looks as if your card is recognized by the system. It may be a NetworkManager issue. I have a page about troubleshooting wireless at https://srobb.net/wireless.html. You'll also want the wireless-tools package. Both of these are probably on the install media. If you ran it from a network install image though, you'll have to download it on another computer. I'm not sure what dependencies either of them have. (Or, as it seems you have an ethernet connection, you can use that to download the net-tools and wireless-tools.

      Not sure about the iw command, I don't use it. But wireless-tools provides the iwlist command that I mention in the linked article, and with that, you can see if your card at least works.

      If this does show the card to be working, also keep in mind that on RH based systems, /etc/resolv.conf is now a symlink to some NetworkManager file so name resolution won't work out of the box. But first, let's see if your card can work.
      New users should check the FAQ and Read Me First pages
